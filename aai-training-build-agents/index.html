The Problem

Agent demos are easy.
Reliable agents need a lifecycle.

The first version of an agent can look impressive after a few prompt edits. The hard part is making behavior stable, testable, reviewable, and safe to change as the agent grows.

Prompt drift

Instructions accrete contradictions and hidden assumptions.

Tool ambiguity

The model sees tools but cannot reliably choose or parameterize them.

Eval afterthoughts

Tests get written after behavior exists, so they codify accidents.

Local/platform drift

Nobody knows whether source of truth is UI, local files, or last push.

Debug by vibes

Teams change prompts until one example passes, then regress another flow.

TDD Contract

Before the agent exists, define how it should behave.

The build workflow is test-driven. The skill first turns the PRD and source artifacts into a TDD: architecture, tools, routing, variables, callbacks, test data, and eval coverage.

Best Practices

The skill encodes the practices we want builders to repeat.

01

Agents as code

Pull, edit, lint, push, review, and keep local files as the working surface.

02

TDD as source of truth

The TDD defines intended behavior and eval coverage before implementation.

03

Structured instructions

Use XML-like sections, clear triggers, scoped taskflows, and positive rules.

04

Tools are model-facing UX

Tool names, docstrings, args, and return schemas are instructions to the model.

05

Evidence-driven iteration

Run evals, read failure clusters, diagnose causes, then fix.

Determinism Pattern

Let the LLM understand language.
Let code enforce control flow.

LLM owns

  • Understanding user intent
  • Interpreting natural language
  • Choosing conversational wording
  • Detecting frustration, escalation, issue category

Callbacks / tools own

  • Auth/profile derivation
  • Deterministic escalation
  • Session ending
  • Tool-call execution
  • Slot filling and retry bounds
  • Text-before-end-session guarantees
User intent → state trigger → before_model_callback → deterministic tool/action

Instructions tell the model what to detect. Callbacks decide what must happen.

Eval Contract

Use the right eval for the behavior you need to trust.

ACallback tests

LocalFastest

Pure Python. Validate deterministic callback behavior locally.

BTool tests

IsolatedSchemas

Validate tool behavior, outputs, and failure modes.

CPlatform goldens

LiveDeterministic

Turn-by-turn ideal conversations for fixed routing and callback-enforced paths.

DLocal simulations

Open-endedGoal-based

Variable flows and real-world behavioral checks where conversations can legitimately vary.

Golden vs Sim

Same path every time?
→ Golden
Outcome matters, path varies?
→ Simulation

Goldens test known deterministic paths. Sims test flexible goals.
