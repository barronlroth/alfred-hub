Coding Agent Task · CX Agent Studio

Two competing skill philosophies

A coding agent is trying to build, evaluate, and improve a CX Agent Studio customer-service agent. The open question: should it discover many narrow top-level skills, or one router skill that lazily loads task-specific references?

Strategic read

Use small skills when the workflow is genuinely standalone.

Agent Foundry, eval hill-climbing, and review can be separate if they have different audiences, release cadences, or toolchains. Otherwise, every new workflow increases skill discovery noise for the coding agent.

Likely direction

Use a CXAS router skill for product-specific depth.

  • One top-level trigger: CX Agent Studio
  • Intent-based reference loading
  • Less catalog pressure as enterprise agents reach hundreds of skills
