OpenClaw Memory

Your memory,
on your machine.

QMD replaces cloud-dependent vector search with a fully local pipeline — query expansion, hybrid retrieval, and neural reranking. No API calls. No data leaving your device.

Side by side

Two approaches to the same files

Both systems search the same Markdown files — MEMORY.md and memory/*.md. The difference is how they find what's relevant.

Default · Built-in

Original Memory

  • → Sends each query to OpenAI's API for embedding (text-embedding-3-small by default)
  • → Single-vector cosine similarity matching against pre-embedded chunks
  • → Optional BM25 hybrid search for keyword matching
  • → Requires an API key (OpenAI, Gemini, or Voyage)
  • → Memory content sent to external servers for embedding
  • → Fast, cheap, works everywhere — but cloud-dependent
Experimental · Local-first

QMD Memory

  • → Runs a local 1.7B LLM to expand your query into multiple search angles
  • → Hybrid retrieval: BM25 keywords + vector similarity combined
  • → Neural reranking via a local 0.6B model (Qwen3 Reranker)
  • → No API key needed for memory search — zero cloud dependency
  • → All data stays on-device — nothing leaves your machine
  • → ~2GB disk for models, cached after first run

How QMD searches

Three-stage local pipeline

Stage 01

Query Expansion

A local 1.7B LLM takes your query and generates multiple search variants — lexical, vector, and hypothetical document embeddings. One question becomes five angles of attack.

›
Stage 02

Hybrid Retrieval

Each expanded query runs against both BM25 (keyword) and vector (semantic) indexes simultaneously. Candidates are pooled and deduplicated by chunk ID.

›
Stage 03

Neural Reranking

A local 0.6B Qwen3 reranker scores every candidate for true relevance. The top results surface with confidence scores — no cloud round-trip needed.

Setup

Three commands, one config line

# Install dependencies
brew install oven-sh/bun/bun sqlite
bun install -g https://github.com/tobi/qmd
openclaw.json 
{
  "memory": {
    "backend": "qmd",
    "qmd": {
      "command": "/Users/you/.bun/bin/qmd",  // if not on system PATH
      "sessions": {
        "enabled": true  // optional: index chat history too
      }
    }
  }
}

First search downloads two GGUF models (~1.9GB total). After that, everything runs from cache. If QMD ever fails, OpenClaw automatically falls back to the built-in vector search — no downtime.
